## Root Cause

`crm/Customers.tsx` and `crm/Regions.tsx` were full 164-line copies of the old `View.tsx` with state-based modal code. The route system (`_app.$moduleId.$table_name.tsx`) loads specific components first (e.g., `views/crm/Customers.tsx`) before falling back to the generic `views/View.tsx`. The previous commit only fixed `View.tsx`, which was never loaded for `/crm/customers`.

## Fix

Replaced both specific view files with re-exports from the generic `View.tsx`:
```tsx
export { View } from '../View'
```

The generic `View.tsx` (fixed in the first commit) uses URL-driven modal/sheet state — both sidebar and modal navigate to update the URL, creating deep links.
