## Changes

### 1. `Header.tsx` — fix breadcrumb overlap when sidebar collapses

The `Header` component wrapped its content in `<header className="sticky top-0 z-50 ... h-16">`. This inner sticky header lived inside the AppLayout outer `<header>` which transitions from `h-16` → `h-12` when the sidebar collapses to icon-only mode.

When collapsed, the outer header occupied only 48 px, but the inner sticky header extended 64 px, covering the first 16 px of page content — exactly where the breadcrumb sits. Replacing the inner sticky `<header>` with a `<div className="flex h-full ...">` fixes the overlap: the content fills the outer header's height regardless of collapse state.

### 2. `DataTableView.tsx` — `key={tableName}` on `DataTableRoot`

When navigating between entities (e.g. Entities → Fields → Modules), React reused the same `DataTableRoot` instance. The niko-table internals (`useGeneratedOptions`) still referenced column IDs from the previous entity. Calling `row.getValue('module_id')` on a table that no longer has that column produced `[Table] Column with id 'module_id' does not exist` errors on every navigation.

Adding `key={tableName}` forces React to fully unmount and remount the table when the entity changes, starting with a clean slate.

### 3. `DataTableView.tsx` — cell text truncation

Text and reference-label cells now render with `max-w-[200px] truncate` and a `title` attribute so:
- Long strings are capped at 200 px and show `…` (visible in the Description column above)
- Full text is accessible via hover tooltip
- Badge (enum/boolean) and numeric cells are unchanged — they're naturally short
